## Adding animations
problem with adding animations is that EVERYTHING has to be animated/smooth. you can't just like animate card distribution. 

you spend a while making this all look slick and for it to 'pop'.

but now you notice that when cards are added to a hand, it looks a bit janky. how to solve this?

you then have to animate cards being added to the pile. like they should fade in instead of just 'jump'

maybe this is where a natural animator/designer naturally sees these ahead of time, like a coder. maybe this whole post reads to a designer how tech people feel when looking at coders. "like, DUH!"

with coding, being competent, you can see things ahead of time. like 'add a log in button' means you instantly start thinking about databases, schema design, auth, api etc.

but yeah it was a surprise for me how 'all or nothing' this is. a game with no animation looks and feels better than a game with half-baked animations. 

(and yes, this is why selfishly the default is for animations to be on! they took a lot of time to implement!)
